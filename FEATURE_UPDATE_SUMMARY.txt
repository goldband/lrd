📌 WHAT WAS ADDED

A new scheduled notification that runs EVERY FRIDAY AT 8:00 PM showing:

✅ Complete list of confirmed players for Saturday & Sunday games
✅ Chronological order (who confirmed first is listed first)
✅ Exact timestamp when each player confirmed (to the second)
✅ Total player count per game
✅ Game details (date, time, venue)
✅ Personalized for confirmed vs. unconfirmed players
✅ Creates both email and in-app notifications

📊 HOW IT WORKS

EXAMPLE WORKFLOW:

Thursday 10 AM - Sarah confirms for Saturday game
  └─ Timestamp: Apr 10, 10:00:00 AM

Thursday 2 PM - Mike confirms for Saturday game
  └─ Timestamp: Apr 10, 2:15:00 PM

Friday 8 AM - Emily confirms for Saturday game
  └─ Timestamp: Apr 11, 8:30:00 AM

Friday 8:00 PM - SCHEDULED TASK RUNS
  ├─ sendFridayConfirmedPlayersSummary() executes
  ├─ Retrieves all games for Saturday & Sunday
  ├─ Gets confirmed players for each game
  ├─ Orders by timestamp (earliest first)
  ├─ For each player:
  │  ├─ Creates in-app notification
  │  ├─ Generates personalized email
  │  └─ Sends via SMTP
  └─ All emails delivered

🔧 TECHNICAL DETAILS

FUNCTION NAME
sendFridayConfirmedPlayersSummary()

LOCATION
server.js (lines ~480-640)

CRON SCHEDULE
0 20 * * 5  (Friday at 8:00 PM UTC)

QUERY USED
SELECT u.id, u.name, a.updated_at 
FROM availability a
JOIN users u ON a.player_id = u.id
WHERE a.game_id = ? AND a.status = 'yes'
ORDER BY a.updated_at ASC

DATABASE
Uses existing tables:
  - games (date, time, venue)
  - users (name, email)
  - availability (status, updated_at ← TIMESTAMP!)
  - notifications (type, title, message)

NOTIFICATION TYPE
'weekend_summary'

   To: All players
